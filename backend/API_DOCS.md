# 🚀 AI Trading Platform - API Documentation

This document provides the technical specification for the SaaS trading backend.

## 📌 Base Configuration
- **Base URL:** `http://localhost:3000`
- **Authentication:** Bearer JWT Token required for all routes except signup/login.
- **Content-Type:** `application/json`

---

## 🔐 Authentication
Protected routes require the header: `Authorization: Bearer <token>`

### 1. User Signup
`POST /auth/signup`
- **Request Body:**
  ```json
  { "email": "user@example.com", "password": "securepassword" }
  ```
- **Success Response:** `201 Created`

### 2. User Login
`POST /auth/login`
- **Request Body:**
  ```json
  { "email": "user@example.com", "password": "securepassword" }
  ```
- **Success Response:** `200 OK`
  ```json
  { "token": "eyJhbG...", "user": { "id": 1, "email": "user@example.com" } }
  ```

### 3. Get Current User (Me)
`GET /auth/me`
- **Success Response:** User object with balance and telegram connection status.

## 📈 Charting & Market Data

### 1. Get OHLCV History
`GET /charts/:symbol`
- **Example:** `GET /charts/BTC/USDT`
- **Success Response:**
  ```json
  {
    "symbol": "BTC/USDT",
    "data": [
      [timestamp, open, high, low, close, volume],
      ...
    ]
  }
  ```

## 🛠 Strategy Management

### 1. List Available Strategies
`GET /strategies`
- Returns all 3 system strategies (GridMeanReversion, TrendFollower, UltimateFuturesScalper).

### 2. Subscribe to Strategy
`POST /strategies/subscribe`
- **Request Body:** `{ "strategyId": 3 }`

### 3. Start Strategy Engine
`POST /strategies/run`
- **Request Body:** `{ "strategyId": 3 }`
- **Note:** Starts the signal generation engine for that strategy.

### 4. Get Strategy Signals
`GET /strategies/:id/signals`
- Returns the 50 most recent trade signals generated by the strategy.

---

## 🤖 Telegram Integration

### 1. Connect Bot
`POST /telegram/connect`
- **Request Body:** `{ "telegramId": "12345678" }`
- **Note:** Links the user account to a Telegram chat ID for signal alerts.
