# Verification Context

## Codebase Changes Applied
1. **signalEngine.js**: 
   - Fixed `result.lastID` to `result.id` so `live_orders.signal_id` will be populated correctly instead of being null.
   - Added a global check `SELECT id, strategyId, side, price FROM signals WHERE symbol = ? AND status = 'active' LIMIT 1`. This ensures that if a symbol is already active, no other strategy (or the same strategy) can open a new trade on it.
2. **liveTradeOrchestrator.js**:
   - `_processSignalForAccount` exit logic updated to first find open orders using `o.signal_id === signalId`, with a fallback to `symbol` matching for old legacy trades.
   - `_startOrderWatcher` now selects all `OPEN`, `NEW`, `PARTIALLY_FILLED` orders (removed the `order_type='limit'` restriction) to catch any type of stuck trade and clean them up if they are older than 5 minutes.
3. **app.js**:
   - In `/live-trading/dashboard`, fixed the state reconciliation bug. It now only forces a DB order to `CLOSED` if `amt === 0` AND the DB order status is already `FILLED`. It will no longer prematurely kill `OPEN` limit orders.
